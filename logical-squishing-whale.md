# 基金多维度分析平台 — 产品方案

## 背景与目标

投资者面对市场上数千只基金，往往只能看到简单的涨跌幅和排名，缺乏深入的多维度分析工具。本平台旨在提供一站式的基金数据分析服务，帮助用户从收益、风险、估值、定投等多个角度全面评估一只基金。

## 核心需求

- **任意基金**：支持按代码或名称搜索任意基金（不仅限于 QDII）
- **多维度指标**：年化收益、最大回撤、波动率、夏普比率、PE 估值、定投回测
- **可视化**：净值走势图、回撤曲线图、定投过程图
- **教育性**：指标解读弹窗、投资大师风格点评，帮助用户理解指标含义
- **交互方式**：Web SPA，浏览器打开即用，首页展示平台介绍，搜索后进入数据分析面板
- **数据源**：东方财富公开接口（免费、国内稳定、无需 API Key）

## 技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端 | 原生 JavaScript SPA | 无框架依赖，轻量快速 |
| 图表 | Chart.js 4.4 | 成熟稳定，支持多种图表类型 |
| 后端 | Python + FastAPI | 金融数据处理生态好，东财接口调用方便 |
| 数据库 | SQLite + SQLAlchemy | 数据量小，无需部署数据库服务 |
| 数据抓取 | httpx | 轻量异步 HTTP 客户端 |

## 数据源设计

### 1. QDII 基金列表
- **接口**：`fund.eastmoney.com/data/rankhandler.aspx`（筛选 QDII 类型）
- **内容**：基金代码、名称、规模、跟踪指数
- **更新频率**：按需拉取（用户搜索时触发）

### 2. 历史净值
- **接口**：`fund.eastmoney.com/pingzhongdata/{code}.js`
- **内容**：单位净值时间序列（`Data_netWorthTrend`）+ 累计净值（`Data_ACWorthTrend`）
- **更新频率**：按需拉取（首次访问某基金时入库，后续从 DB 读取）

### 3. 前十大持仓
- **接口**：`fundf10.eastmoney.com/FundArchivesDatas.aspx`
- **内容**：股票代码、名称、权重、报告期
- **更新频率**：按需拉取（每只基金每个 session 只拉取一次）

### 4. 股票 PE 数据
- **接口**：`push2.eastmoney.com/api/qt/stock/get`
- **内容**：个股 PE TTM（市盈率）
- **更新频率**：每次调用 `/api/funds/{code}/metrics` 时实时拉取

## 核心计算模型

### 年化收益 (CAGR)
```
年化收益率 = (期末净值 / 期初净值) ^ (1 / 年数) - 1
```

### 最大回撤 (Max Drawdown)
```
最大回撤 = min( (当日净值 - 前期最高净值) / 前期最高净值 )
从净值序列中找出历史最大跌幅，含峰值日期和谷底日期
```

### 波动率 (Volatility)
```
日收益率标准差 × sqrt(252)
其中日收益率 = ln(今日净值 / 昨日净值)
```

### 夏普比率 (Sharpe Ratio)
```
(年化收益 - 无风险利率) / 波动率
无风险利率按中国 10 年期国债收益率 ≈ 2.5%
```

### 索提诺比率 (Sortino Ratio)
```
(年化收益 - 无风险利率) / 下行波动率
下行波动率仅统计日收益 < 0 的数据
```

### 卡玛比率 (Calmar Ratio)
```
年化收益 / |最大回撤|
衡量每单位最大亏损能带来的收益
```

### 胜率 (Win Rate)
```
收益为正的天数 / 总交易天数 × 100%
```

### PE 比率
```
基金 PE = 1 / Σ( 持仓权重_i / 个股PE_i )
使用调和平均值（更能反映组合估值），要求覆盖权重 ≥ 50%
```

### 定投回测 (DCA Backtest)
```
每月首个交易日投入固定金额
累计份额 = Σ( 每月投入 / 当月净值 )
最终市值 = 累计份额 × 最新净值
总收益率 = (最终市值 - 总投入) / 总投入 × 100%
同时计算一次性投入（Lump Sum）作为对比
```

## 后端 API 设计

| 接口 | 方法 | 说明 |
|------|------|------|

| `/api/funds/{code}/dca?amount=&years=` | GET | 定投回测 |
| `/api/funds/{code}/commentary` | GET | 投资大师点评 |

## 数据库表设计

### funds 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| code | VARCHAR(10) | 基金代码，唯一索引 |
| name | VARCHAR(100) | 基金名称 |
| fund_type | VARCHAR(50) | 基金类型（默认 "QDII"） |
| tracking_index | VARCHAR(100) | 跟踪指数 |
| scale | FLOAT | 基金规模（亿元） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### holdings 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| fund_id | INTEGER FK | 关联 funds.id |
| stock_code | VARCHAR(20) | 股票代码 |
| stock_name | VARCHAR(100) | 股票名称 |
| weight | FLOAT | 持仓权重（%） |
| report_date | DATE | 季报日期 |
| created_at | DATETIME | 抓取时间 |

### fund_nav 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| fund_id | INTEGER FK | 关联 funds.id |
| nav_date | DATE | 净值日期 |
| unit_nav | FLOAT | 单位净值 |
| cumulative_nav | FLOAT | 累计净值 |
| daily_return | FLOAT | 日收益率（入库时预计算） |

唯一约束：`(fund_id, nav_date)`

## 已知限制

| 限制 | 影响 | 后续方向 |
|------|------|----------|
| 仅自动发现 QDII 基金 | 非 QDII 基金需通过搜索触发生效 | 扩展基金类型支持 |
| 持仓仅前十大 | PE 计算存在偏差 | 若数据源支持，获取完整持仓 |
| 季报滞后 1-2 月 | 基金经理调仓后 PE 产生偏差 | 提示用户报告日期 |
| 美股 PE 覆盖不全 | 部分股票无法获取 PE | 标记覆盖率，低于 50% 不显示 |
| 不支持基金对比 | 只能逐只看 | 后续添加多选对比功能 |
